# Mission: SDE3 / Staff Engineer Interview Readiness

## Why
Currently SDE2 (Walmart, ~2 yrs experience). Targeting SDE3 and Staff-level roles (FAANG tier + high-bar Unicorns). The 2-month window before active interviewing is the entire runway. The cost of failing a Staff loop is 6+ months of wasted comp trajectory, so the goal is to walk into any loop confident the rubric is covered.

## Success looks like
- **System design round** (45 min): Drive a Staff-caliber design from requirements → high-level → deep dive → cross-cutting concerns → tradeoffs. Hit depth on 2-3 areas, breadth on the rest. Quantify back-of-envelope numbers. Name real-world systems (Kafka, Cassandra, Spanner) when relevant.
- **Low-level design round** (45 min): Ship a working OOD design for a real-world system (URL shortener, rate limiter, ride-dispatch) with clean class boundaries, design patterns applied correctly, and tradeoff justification.
- **Coding round** (45 min): Solve medium-hard LeetCode in 25 min, hard in 35 min, in Java, with clean variable names and the patterns I reach for under pressure (Sliding Window, Two Pointers, BFS/DFS, DP, Graph, Heap, etc.).
- **Behavioral + leadership signals** for Staff: tradeoffs under ambiguity, mentoring/teaching signals, project scoping, conflict resolution, technical influence across teams.

## Constraints
- **Time budget**: ~1.5 hrs/day weekdays, 3-4 hrs/day weekends = ~12 hrs/week. Total runway = 2 months × 4.3 weeks × 12 hrs = ~103 hrs.
- **Stack**: Java primary (per user profile), Python secondary. Frameworks: Spring Boot, Dropwizard, Kafka, AWS, K8s — already strong here; leverage as concrete examples.
- **Existing assets**: 30+ System Design articles already authored in Obsidian vault (URL Shortener → K8s, etc.). Published at abster12.github.io/notes/. These are the *source material* — lessons extract, compress, and test against them. Do NOT rewrite.
- **Cost-conscious**: Free resources first (DDIA excerpts, Alex Xu YouTube, free tier tools). Paid only if it unlocks something a free source doesn't.
- **Java patterns**: Coding interview prep folder in vault has 18 pattern-based articles with Java solutions — use as primary coding reference.
- **Format preference** (from NOTES-style memory): Articles = paragraphs for concepts, bullets ONLY for lists/steps, ASCII diagrams for arch/flow, tables for comparisons. No walls of text. Likes parallel batch processing for large tasks.
- **Scope discipline**: User has been visibly frustrated when scope was violated on resume work. Do ONLY what's asked. If a question comes up mid-lesson that needs a tangent, flag it and continue, don't derail.
- **No screenshots of web pages** unless explicitly asked.

## Out of scope (for the 2-month window)
- Mobile/iOS-specific design patterns
- ML system design specifics beyond the standard "design ML pipeline" question
- Frontend engineering depth (React internals, etc.)
- New language acquisition (Rust, Go) — even if some Staff roles ask for them
- Behavioral round prep beyond ~2 dedicated sessions (most weight is technical for SDE3/Staff loops)
- Resume/LinkedIn updates (separate workstream)
- Building a portfolio project (already have Stepwise + advisor-executor-strategy + notes-site)
